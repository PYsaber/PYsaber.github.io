docker中使用GPU版MXNet

发表于 2017-06-14 | 分类于 生产环境 | | 阅读次数

实验室有自己的GPU集群，跑起实验来比自己电脑快很多，于是准备把工作环境移到实验室的服务器上，但是在配置环境的过程中遇到了很多问题，主要是docker中使用GPU的问题，网上基本查不到，都是实验室师兄们总结出的经验，在这里记录一下。

为什么使用Docker？

Docker可以理解为一个轻量级的命令式虚拟机，我认为使用它的目的主要有两个：

  1. 便于移植环境：例如，你编写了一个python应用，其中用到了许多python的扩展库，当你要将你的应用移植到另一台电脑时，还需要在另一台电脑上配置各种环境，配环境的过程中可能还会遇到很多问题。使用Docker，你只需要将你的docker镜像拷贝到另一台电脑上即可。
  2. 程序版本管理，防止冲突：当你的应用越来越多时，可能会遇到各种程序需要同一个程序的不同版本，特别是在公用的服务器上，很容易出现这种问题。我们实验室基本是3,4个人共用一台服务器，在我用的这台服务器上一位师兄用的MXNet版本是很早之前的0.7.0版本。现在MXNet官网已经更新到了0.10.0版本了，很多语法已经不通用了，所以我才想着装一个docker然后在docket里安装最新版的来避免环境冲突。

    如何安装?

    其实MXNet官网的安装说明写得挺详细的，还提供了配置好了GPU版MXNet的docker镜像，如果你的设备可以任由你更改环境，那么基本不会出现问题，只有一步和官网说明不同，在建立docker容器的时候需要加上一些参数设置:--device /dev/nvidiactl --device /dev/nvidia-uvm --device /dev/nvidia0 --device /dev/nvidia1,这些参数的目的是挂载显卡到docker，官网中的命令没有写上这些参数，可能会导致很多莫名奇妙的问题。

    GPU驱动不能更改情况下如何在docker中使用GPU？

    由于服务器是公用的，所以我不能乱改上面显卡驱动，这里需要理解NVIDIA显卡的两个概念，driver version和runtime version。driver version是直接装在显卡里的驱动，而runtime version则是安装在你的操作系统里来掉用GPU的。当这两个版本不同就会出现问题，由于我不能更改服务器的显卡驱动版本，在使用MXNet官网提供的docker镜像时就出现了以下问题：CUDA driver version is insufficient for CUDA runtime version。报错信息说的很清楚，driver version和runtime version 不同，只要知道这两个东西指的是什么，就不难理解为什么会报这个错。前面说了，driver version指的是显卡中的驱动版本，这个是跟随硬件本身的，而runtime version是显卡运行时环境的版本，是装在操作系统里的。所以你移植你的docker镜像只能移走runtime环境。所以，报这个错的原因就是我使用的服务器上的显卡驱动版本和MXNet提供的docker镜像中安装的显卡runtime版本不同，由于不能更改服务器环境，所以我的解决方案是更改MXNet提供的docker镜像中的runtime版本，如果你可以更改硬件环境，你也可以将驱动版本改为和镜像中的runtime版本一致。查看驱动版本使用nvidia-smi命令: 如上图，可以看到我的显卡驱动版本为367.48,去NVIDIA官网下载对应版本完整驱动(runfile)，我下载的是cuda_8.0.44_linux.run，然后使用sh cuda_8.0.44_linux.run --noexec --target pysaber命令解压缩runfile，找到里面一个名为NVIDIA-Linux-x86_64-367.48.run的文件，这个安装文件就是runtime环境安装包，在MXNet提供的镜像中安装一遍即可。
    我已经将我配置好的镜像上传到了docker hub,你们可以直接pull这个版本然后更改其中的runtime版本为你的驱动版本即可。
